# Stage-3 Universal Exploit Validation Engine

This repository represents **Stage-3** of an automated, LLM-driven smart contract exploit validation pipeline. It acts as a deterministic "Fact-Checker," executing candidate exploits generated by an LLM in a rigid EVM environment to verify their success and natively prove vulnerabilities.

---

## 🚀 The Pipeline Architecture

The pipeline consists of three core components:

1. **The Attempt Spec (`attempt_*.json`)**  
   A declarative representation of an exploit attempt. It defines a topology of smart contracts to deploy (`setup`), the exact EVM state parameters required (`prestate`), the ordered execution steps (`calls`), and strictly typed definitions of exploit success (`assertions`).

2. **The Dynamic Generator (`generator.py`)**  
   The translation layer. It ingests the generalized Attempt Spec and dynamically compiles a mathematically sound Foundry Test (`ExploitTest.t.sol`). It features an advanced alias resolver that handles cross-contract interactions seamlessly, eliminating the need to hardcode specific protocols or attacker logic mappings natively.

3. **The Execution Driver (`driver.py`)**  
   The execution layer. It programmatically triggers the Foundry environment, capturing the internal execution metadata, calculating profit margins (`balance_deltas`), aggregating multi-level contract traces, and formulating the final `Fact-Check Report` mapping. This report is subsequently routed back to the Crew Boss for classification (Verification or Retries).

---

## 🛡️ Supported Vulnerability Validations
The V2 Universal Engine was rebuilt to be perfectly **Contract Agnostic**, meaning it is able to validate practically any vulnerability class natively:
- **Reentrancy Loops** (Automatic EVM underflow detection & recursive fallback evaluations)
- **Access Control & Phishing** (Generic `tx.origin` and `msg.sender` manipulation tracing)
- **Arbitrary Proxy Delegates** (Detects storage collisions & logic injections via `storage_eq`)
- **Denial of Service (DoS)** (Validates state blockers utilizing EVM `call_reverts` checks)
- **Temporal & Invariant Flaws** (Simulates precise state distortions natively using `vm.warp` and `vm.roll`)

## ⚙️ How to Run

### 1. Prerequisites
- [Foundry](https://getfoundry.sh/) (`forge` installed in your PATH)
- Python 3.8+
- The `python-dotenv` package:
  ```bash
  pip install python-dotenv
  ```

### 2. Execution
Run the driver script via Python. The V2 module comes with an interactive CLI allowing you to seamlessly benchmark the 14-file Vulnerability Test Dataset included natively in the directory.

```bash
py driver.py
```

### 3. Interactive Selection
Instead of swapping JSON files manually, the CLI will automatically parse all `.json` files starting with `attempt_` in the directory. 
Simply type the number corresponding to the vulnerability Attempt Spec you wish to validate, and hit Enter.

```bash
=== Available Exploit Attempts ===
[1] attempt_accesscontrol.json
[2] attempt_delegatecall.json
[3] attempt_dos.json
...
Select an attempt file to run (enter number): 5
```

The system will build the isolated testing topology, enforce the LLM's payload, and output the ultimate `Fact-Check Report` and the `Crew Boss Decision`.
